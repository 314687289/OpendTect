OpendTect V5.0.x : Release info

----
v5.0.6: March 5, 2015
 * Fix for crash while generating Depth/Time model in
   'Tie well to Seismic' window
 * Fix for defining constants with capital 'C' in Mathematics attribute
 * Fix for unit conversion and importing well track in 'Manage Wells' window
 * Fix for picking fault manually on time slice
 * Number of CPUs allocated to OpendTect is displayed in the status bar only
   if specified
 * Merging of two or more Steering Cubes now results in another Steering Cube
 * Fix for misleading warning messages while overwriting a horizon
   after gridding
 * 'Save' button on the horizon tracking toolbar is now deactivated if the
   horizon is already saved
 * Fix for markers not being listed in cross-plot window
 * Fix for error message in Attribute versus Well Data cross-plot window
   at start-up
 * Improved 'Display only at sections' for horizons
 * Fix for Wavelet import
 * After multi-machine processing the temporary data folder is now removed
 * Fix for attribute processing on 2D lines with trace number steps of
   more than 1

v5.0.5: February 12, 2015
 * Made the 'Allow spin animation' option in Scene properties window work
 * Added 'Delete rows' option in well log editor
 * Implemented copy/paste functionality from OpendTect tables to external
   spreadsheets and vice-versa
 * Improved synchronization of cursor position between scenes
 * Improved ASCII import of Well time-depth models
 * Improvements in well track import
 * Fix for crash while evaluating Mathematics attribute
 * Fix for crash while creating seismic cube with selected depth range using
   volume builder tool
 * Fix for crash while trying to save horizon without tracking anything
 * Fix for crash while importing Seismic data from simple file
 * Fix for displaying seed based iso-surface volume in scene
 * Fix for displaying of connection line between previous node and new node
   while picking fault plane
 * Fix for exporting of prestack data store into a SEGY
 * Fix for displaying of correct log units in 'Tie well to seismics' window
 * Fix for change of active color bar after saving active color bar with a
   new name
 * Fix for Splicing two logs using Mathematics engine in "Calculate new logs"
   window in well tie module
 * Fix for well name size not keeping up with snapshot resolution
 * Removed the 'Retrack all' button from Horizon tracking toolbar
 * Fix for displaying 2D lines with missing traces in 2D viewer
 * Fix for missing Random line names in restored sessions
 * Fix for tiled display of Horizon after changing Z Scale
 * Fix for creating Implicit body objects
 * Data from/to Simple binary files can be swapped during import/export by
   setting the environment variable OD_SIMPLE_FILE_BINARY_SWAPPED to 'Yes'.
   This allows reading/writing of (big endian) image files on windows
   and Linux workstations


v5.0.4: January 15, 2015
 * Added 'Enable Virtual Keyboard' option in OpendTect Settings
   window (under the 'General' tab)
 * Random line menu option 'Edit nodes' is renamed to 'Positions'
 * Fix for crash while changing Z-scale during horizon tracking
   using track from seed option
 * Fix for crash when switching to view mode while editing a fault
 * Fix for deletion of current survey when aborting new survey creation
 * Fix for crash while closing OpendTect main window after changing markers
   size from 'Well display properties' window
 * Newly created 2D lines from 'Create 2D grid' tool can now be displayed
   without restarting OpendTect
 * Fix for appearance of occasional unsorted traces in exported 2D SEGY files
 * Color settings can now be saved/retrieved per 2D Seismic dataset
 * Removed 'Rename line' button from Manage 2D Seismics as renaming can and
   should be done via 'Manage 2D Geometry'
 * Fix for incorrect/non- opening of user documentation on certain KDE systems
 * Fix for undesired entries in the Survey .omf file for imported SEGY files
 * Fix to ensure correct updating of the survey map on pressing 'Apply' in
  'Edit survey parameters' window
 * Fix for errors while reading header files of results during
   Madagascar processing
 * Fix for displaying title-box in GMT maps
 * Fix for disappearance of volume box from the scene while dragging in
   certain surveys
 * Separated shading settings for Volume rendering and Surface rendering
 * Fix for crash while changing gather display in case of 2D Seismic
 * Fix for closing 'Please specify' window while loading default cube
   when no default cube is set

v5.0.3: December 11, 2014
 * Fix for crash while tracking 2D horizon
 * Fix for crash while copying wavelet from another survey
 * Fix for crash while creating seismic output between 2D Horizons
   when no line has been selected by the user
 * Fix for crash while restoring session with Faultsticksets
 * Fix for import of RokDoc Probability Density Function when one of the two
   dimensions is equal to 1. Now such function are imported as 1D functions
 * Fix for display of Faults and Faultsticksets using 'Only at sections'
   option not honouring the Z range of the section
 * Fix for distorted vertical position of Top/Bottom images while changing
   Z-scale of the scene
 * Fix for displaying certain types of Geobodies
 * Fix for color bar annotation color not getting updated immediately after
   changing it in scene properties window
 * Made simple file import of 2D Seismics work again
 * Fix for wrong unit displayed in the table header of Edit Well Markers
   window while toggling 'Z in feet'
 * Enabled displaying of X,Y,Z and trace number on status bar
   in 2D viewer for 2D data
 * Fix for X coordinate values duplicated as Y coordinates while
   exporting 2D Geometry
 * Fix for jumping scroll bar of the Marker list while checking/unchecking
   markers in the well display properties window
 * Made 'Examine' button work in Import windows on MAC platform
 * Fix for 'Pick' mode not getting automatically turned on after adding a new
   FaultStickSet to the tree
 * Fix for a 3D cube entry being created in the database while saving a Random
   line as a 2D line
 * Implemented 'Show all', 'Hide all' and 'Remove all' options for
   items in 2D viewer

v5.0.2: November 17, 2014
 * Fix for displaying contours on 3D horizon
 * Fix for displaying of volume attribute data
 * Fix for calculating mathematics attribute with constants
 * A Few fixes for Zoom wheel
 * Fix for color-blended horizons
 * Fix for setting up new survey manually
 * Fix for saving and restoring top/bottom image in sessions
 * Added warning messages if the input logs for synthetic-to-seismic matching
   have incorrect values
 * Fix for crash while creating 3D cube from 2d data
 * Fix for wrong bin size displayed in survey selection window
 * Fix for default value of center sample while importing a wavelet
 * Fix for crash while importing Horizon
 * Fix for crash while displaying well log attributes

v5.0.1: October 28, 2014
 * Visualization is based on OpenSceneGraph (a different graphics library)
 * Rework of the 2D geometry
 * All batch processing can be restarted
 * Attributes on pre-stack gathers
 * Improved horizon-based painter
 * F-K plots on slices
 * Re-worked mathematics expressions
 * Directional texture attributes - Plugin from Joanneum Institute
 * Improved setup for distributed computing
 * Improved handling multi-component / pre-stack interaction
 * Documentation in MadCap Flare: HTML5 output, Context-Sensitive Help, fully
   searchable index and permanently visible TOC
